Phần 3

Lập Bản đồ — SLAM Toolbox

Thuật toán Graph-based SLAM

Nguyên lý: Xây dựng đồ thị tư thế (pose graph), mỗi node là vị trí robot, mỗi cạnh là ràng buộc từ scan matching.

Scan Matching — Correlative

  • Xây bản đồ xác suất + Gaussian blur
  • Quét thô: duyệt lưới (dx, dy, dθ)
  • Tối ưu tinh: Ceres Solver (Levenberg-Marquardt)
E(T) = Σᵢ [1 − M(T·pᵢ)]²

M(x) = xác suất vật cản tại x. Khi T chính xác → E(T) → 0

Loop Closure — Đóng Vòng Lặp

Khi robot quay lại vùng đã quét, thuật toán phát hiện sự trùng lặp và hiệu chỉnh sai số tích lũy trong toàn bộ đồ thị.

Quy trình 2 giai đoạn:

  • Coarse matching: chain_size ≥ 10, response ≥ 0.35
  • Fine matching: response ≥ 0.45
  • Tối ưu toàn cục phân bố sai số

Bản đồ Occupancy Grid:

  • Độ phân giải: 0.05 m/pixel
  • Giá trị ô: 0 (trống) → 100 (vật cản), -1 (chưa biết)
  • Cập nhật mỗi 5 giây

Phần 4

Khám Phá Tự Động — Frontier Exploration

Thuật toán Frontier-based Exploration

Frontier = biên giới giữa vùng đã quét (free) và vùng chưa biết (unknown).

Hàm chọn Frontier tối ưu:

score(f) = 0.8·size(f) − 5.0·cost(f)
  • size(f) — kích thước frontier (ưu tiên lớn)
  • cost(f) — chi phí đi đến (ưu tiên gần)
  • min_frontier_size: 0.35m
  • progress_timeout: 45s → blacklist nếu kẹt

Dừng khi: không còn frontier ≥ 0.35m → bản đồ hoàn chỉnh.

Phần 5

Điều Hướng — Nav2 Stack

Định vị — AMCL (Particle Filter)

Adaptive Monte Carlo Localization — ước lượng vị trí robot trên bản đồ bằng tập hợp "hạt" (particles).

Chu trình:

  • Dự đoán: Hạt di chuyển theo odometry + nhiễu
  • Cập nhật: So LiDAR scan với bản đồ → trọng số
  • Lấy mẫu lại: Loại hạt sai, nhân bản hạt đúng
  • Ước lượng: Trung bình có trọng số → vị trí

500–2000 hạt tự điều chỉnh (KL-divergence)

Sensor model: likelihood_field, max 180 tia

Lập Đường đi & Bám Đường

A* — Planner toàn cục

f(n) = g(n) + h(n)
  • g(n) = chi phí thực từ start đến n
  • h(n) = heuristic (Euclidean) đến goal
  • tolerance: 0.5m, allow_unknown: true

DWB — Controller cục bộ

  • Dynamic Window: tập vận tốc (v, ω) khả thi
  • Mô phỏng 1600 quỹ đạo trong 1.2s
  • Critics: PathAlign(20), GoalDist(16), Obstacle(0.05), RotateToGoal(32)
  • max_vel: 0.3 m/s, max_ω: 1.0 rad/s

Phần 6

Theo Dõi Người — Person Tracking

Phát hiện & Nhận dạng Người

Pipeline xử lý:

  • YOLOv8n: Phát hiện thân người (one-stage, conf ≥ 0.5)
  • InsightFace buffalo_l: Trích xuất embedding khuôn mặt 512-d
  • Cosine Similarity: So khớp với database đã đăng ký
cos(θ) = A · B    (|A|=|B|=1)

Ngưỡng nhận dạng: ≥ 0.6

Sensor Fusion — Camera + LiDAR:

  • Camera → bearing (góc) qua pinhole model + TF2
  • LiDAR → leg clustering (0.05–0.30m) + pair (15–35cm)
  • Ghép bearing ↔ leg-pair (tolerance: 0.10 rad)

Điều khiển Bám Theo — PID 3 Tầng

u(t) = Kp·e + Ki·∫e·dt + Kd·de/dt
TầngMục đíchTần số
Servo PIDGiữ target giữa khung hình50 Hz
Wheel YawQuay body khi servo > 30°10 Hz
Linear PIDGiữ khoảng cách 1.0–1.5m10 Hz

FSM Tìm kiếm khi mất target:

  • SEARCH_CONTINUE (0.5s) → tiếp tục hướng cũ
  • SEARCH_SCAN (2.0s) → quét servo trái-phải
  • SEARCH_ROTATE (5.0s) → quay body 360°
  • IDLE → dừng hoàn toàn

Kết luận & Hướng Phát triển

Kết quả đạt được:

  • Robot hoạt động ổn định với SLAM + Navigation + Person Tracking
  • Kiến trúc 3 tầng: Firmware (ESP32) → Middleware (ROS 2) → Web Dashboard
  • Giao diện web real-time, điều khiển từ xa qua trình duyệt
  • Chi phí phần cứng thấp, phù hợp nền tảng nhúng

Hướng phát triển:

  • Tích hợp SLAM 3D (RGB-D camera)
  • Cải thiện nhận dạng đa người trong môi trường đông đúc
  • Thêm chế độ tuần tra tự động (patrol) theo lịch trình
  • Tối ưu firmware trên ESP32-S3 cho hiệu năng cao hơn
  • Ứng dụng: robot giám sát, robot phục vụ, hỗ trợ người già
