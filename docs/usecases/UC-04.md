## UC-04: Theo dõi và bám đuổi người dựa trên nhận diện khuôn mặt

### Mô tả use case

| Mục đích                       | Người vận hành cần robot tự động bám theo một cá nhân cụ thể trong môi trường (ví dụ: đi theo chủ nhân quanh nhà, theo trợ lý trong kho). | Tiền điều kiện                 | 1. Robot đã bật nguồn và kết nối WiFi <br> 2. Web dashboard đã kết nối rosbridge (status = connected) <br> 3. ESP32-CAM đang stream `/camera/image_raw` và publish `/camera_info` <br> 4. ESP32 main board đang publish `/scan` (LiDAR) và nhận `/cmd_vel`, `/servo_cmd` <br> 5. Có ít nhất một người đã được đăng ký thành công (UC-05) <br> 6. Hệ thống đang ở chế độ Tracking, không bật Manual Override                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |

Người vận hành mở "Manage Persons", chọn một người đã đăng ký và đặt làm mục tiêu <br> 4. Dashboard gọi service `/enrollment/set_target(person_id)` <br> 5. `enrollment_node` cập nhật target trong DB, `person_tracker_node` đọc lại `current_target_id` <br> 6. Hệ thống khớp embedding khuôn mặt với DB, gán `is_target = true` cho người trùng `current_target_id` <br> 9. `person_tracker_node` tính khoảng cách (`range_m`) từ leg-pair LiDAR khớp theo bearing và publish `/tracked_persons` (TrackedPersonArray) <br> 10. `tracking_controller_node` nhận target, chạy PID servo (giữ bearing ≈ 0), PID yaw (handoff khi servo vượt ngưỡng), PID linear (giữ khoảng cách trong [target_distance_min, target_distance_max]) <br> 11. Hệ thống publish `/cmd_vel` đến ESP32 và `/servo_cmd` đến servo `camera_pan_joint`, robot quay đầu và di chuyển bám theo người <br> 12. `/tracking_controller/status` được publish 5 Hz để hiển thị trạng thái (TRACKING/SEARCH\_\*), khoảng cách hiện tại và cờ obstacle trên dashboard <br> 13. Khi muốn dừng, người vận hành tắt toggle "Face Tracking" → robot dừng (`/cmd_vel = 0`), controller về trạng thái IDLE |
| Hậu điều kiện (thành công)     | Robot bám sát mục tiêu trong khoảng cách [target_distance_min, target_distance_max], servo + bánh xe phối hợp giữ mục tiêu ở giữa khung hình; trạng thái controller = TRACKING                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| Hậu điều kiện (thất bại)       | Robot dừng tại vị trí an toàn (`/cmd_vel = 0`), controller chuyển về SEARCH\_\* hoặc IDLE; PID state được reset; mục tiêu mất nhưng `current_target_id` vẫn được giữ để sẵn sàng theo dõi lại khi người mục tiêu xuất hiện trong khung hình.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| Xử lý ngoại lệ                 | Mục tiêu rời khung hình ngắn (< 0.5s) → SEARCH_CONTINUE: robot tiếp tục quay theo hướng cũ <br> Mục tiêu mất lâu hơn → SEARCH_SCAN: servo quét trái-phải; nếu vẫn không thấy → SEARCH_ROTATE: robot xoay tại chỗ <br> Vật cản trước robot trong front safety arc → cấm tiến (`linear.x = 0`), vẫn cho xoay <br> `/camera_info` chưa nhận được → bỏ frame, log cảnh báo throttle <br> TF lookup thất bại (camera → laser_link) → bỏ frame, không cập nhật bearing <br> Không tìm được leg-pair khớp bearing → `range_m = NaN`, controller bỏ qua linear command, chỉ chạy servo/yaw <br> Mất kết nối rosbridge → robot dừng (safety timeout firmware)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |

### Lược đồ tuần tự

```plantuml
@startuml UC-04
title UC-04: Theo dõi và bám đuổi người dựa trên nhận diện khuôn mặt

actor "Người vận hành" as Operator
participant "Hệ thống" as System
participant "Person Tracker" as Tracker
participant "Tracking Controller" as Controller

Operator -> System: setMode(TRACKING)
Operator -> System: setTarget(person_id)
System -> Tracker: /enrollment/set_target(person_id)
Tracker --> System: success

Operator -> System: enableFaceTracking(true)

loop Mỗi khung hình camera
    System -> Tracker: /camera/image_raw + /camera_info + /scan
    Tracker -> Tracker: YOLO body + InsightFace embedding + match DB + leg-pair range
    Tracker -> Controller: /tracked_persons (is_target, range_m, bearing_rad)

    alt Mục tiêu xuất hiện
        Controller -> Controller: PID servo + yaw + linear
        Controller -> System: /cmd_vel + /servo_cmd
        System --> Operator: /tracking_controller/status (TRACKING)
    else Mục tiêu mất
        Controller -> Controller: SEARCH_CONTINUE → SCAN → ROTATE
        Controller -> System: /cmd_vel (xoay tìm) + /servo_cmd (quét)
        System --> Operator: /tracking_controller/status (SEARCH_*)
    else Có vật cản phía trước
        Controller -> System: /cmd_vel (linear.x = 0)
        System --> Operator: /tracking_controller/status (obstacle = true)
    end
end

Operator -> System: enableFaceTracking(false)
System -> Controller: trackingEnabled = false
Controller -> System: /cmd_vel = 0, state = IDLE
@enduml
```

### Phân rã thành phần PM

#### Controller: `DashboardWebApp`

- **Nhiệm vụ**: Cho phép người vận hành chọn mục tiêu, bật/tắt tracking, hiển thị
  trạng thái controller real-time và tinh chỉnh PID gain.
- **Service call**: `/enrollment/set_target` (slam_car_interfaces/SetTrackingTarget),
  `/enrollment/list_persons` (slam_car_interfaces/ListPersons),
  `/tracking_controller_node/set_parameters` (rcl_interfaces/SetParameters)
- **Topic publish**: `/cmd_vel` (chỉ khi Manual Override hoặc khi tắt tracking để
  reset về 0)
- **Topic subscribe**: `/tracked_persons` (slam_car_interfaces/TrackedPersonArray),
  `/tracking_controller/status` (std_msgs/String JSON)
- **Input**: Click "Set as target" trên thẻ người → `SetTrackingTargetRequest { person_id }`;
  Toggle Face Tracking → cập nhật state UI
- **Output thành công**: Hiển thị tên mục tiêu, badge trạng thái (TRACKING/SEARCH\_\*),
  range_m, cảnh báo obstacle
- **Output lỗi**: Toast notification khi `/enrollment/set_target` thất bại hoặc
  parse JSON status thất bại

#### UseCase: `PersonTrackingUseCase`

- **Nhiệm vụ**: Orchestrate luồng nhận diện + bám đuổi người mục tiêu.
- **Input**: `person_id: string` (mục tiêu), `enabled: bool` (bật/tắt tracking)
- **Output**: `TrackedPersonArray` (real-time), `TrackingControllerStatus` (5 Hz)
- **Gọi đến**:
  - `rosbridge.callService(/enrollment/set_target)` — đặt mục tiêu
  - `rosbridge.subscribe(/tracked_persons)` — overlay bbox trên camera viewport
  - `rosbridge.subscribe(/tracking_controller/status)` — hiển thị state + range
  - `rosbridge.callService(/tracking_controller_node/set_parameters)` — chỉnh
    9 gain PID (servo/yaw/linear × kp/ki/kd)

#### Node: `person_tracker_node`

- **Nhiệm vụ**: Phát hiện thân người, nhận diện khuôn mặt, fuse với LiDAR
  leg-pair để gán range, publish kết quả tracking.
- **Subscribe**:
  - `/camera/image_raw` (sensor_msgs/Image)
  - `/camera_info` (sensor_msgs/CameraInfo)
  - `/scan` (sensor_msgs/LaserScan)
- **Publish**: `/tracked_persons` (slam_car_interfaces/TrackedPersonArray)
- **Phụ thuộc DB**: `~/.slam_car/face_db.sqlite` — hot-reload embeddings khi DB
  thay đổi (so sánh `db_last_modified`)
- **Tham số chính**: `embedding_threshold`, `leg_cluster_min_width`,
  `leg_cluster_max_width`, `leg_pair_min_gap`, `leg_pair_max_gap`,
  `bearing_match_tolerance`

#### Node: `tracking_controller_node`

- **Nhiệm vụ**: Điều phối servo camera + bánh xe để bám mục tiêu, xử lý các
  trạng thái tìm kiếm khi mất mục tiêu, đảm bảo an toàn trước vật cản.
- **Subscribe**:
  - `/tracked_persons` (slam_car_interfaces/TrackedPersonArray)
  - `/scan` (sensor_msgs/LaserScan) — front safety arc check
- **Publish**:
  - `/cmd_vel` (geometry_msgs/Twist) → ESP32 main board
  - `/servo_cmd` (sensor_msgs/JointState, name = `camera_pan_joint`)
  - `/tracking_controller/status` (std_msgs/String chứa JSON
    `{state, target_id, range_m, obstacle}`)
- **Vòng lặp**:
  - Servo loop 50 Hz — PID servo bám bearing
  - Wheel loop 10 Hz — PID yaw (handoff khi |servo_angle| > servo_handoff_threshold)
    - PID linear (giữ khoảng cách)
  - Lost-target check 10 Hz — chuyển trạng thái IDLE/TRACKING/SEARCH\_\*
  - Status loop 5 Hz — publish JSON status
- **Tham số tunable runtime**: 9 PID gain (`pid_servo_kp/ki/kd`,
  `pid_wheel_yaw_kp/ki/kd`, `pid_linear_kp/ki/kd`)

#### Node: `enrollment_node` _(chỉ phần liên quan UC-04)_

- **Nhiệm vụ**: Quản lý mục tiêu hiện tại trong DB.
- **Service liên quan UC này**:
  - `/enrollment/set_target` (SetTrackingTarget) — cập nhật `current_target_id`
  - `/enrollment/get_target` (GetTrackingTarget) — đọc target hiện tại
  - `/enrollment/list_persons` (ListPersons) — liệt kê danh sách enrolled

#### Firmware: `ESP32 Main`

- **Nhiệm vụ**: Nhận `/cmd_vel` điều khiển động cơ qua TB6612FNG, nhận
  `/servo_cmd` điều khiển servo `camera_pan_joint`, publish `/scan` từ LDS02RR.
- **Subscribe**: `/cmd_vel`, `/servo_cmd`
- **Publish**: `/scan`, `/odom`

#### Firmware: `ESP32 CAM`

- **Nhiệm vụ**: Stream MJPEG, được `cam_bridge_node` chuyển sang `/camera/image_raw`.
- **Publish (qua bridge)**: `/camera/image_raw`, `/camera_info`

#### Lược đồ tuần tự nội bộ PM

```plantuml
@startuml UC-04-internal
title UC-04: Theo dõi người - Internal Sequence

actor "Người vận hành" as Operator
participant "Web Dashboard" as Web
participant "rosbridge" as ROS
participant "enrollment_node" as Enroll
participant "person_tracker_node" as Tracker
participant "tracking_controller_node" as Ctrl
participant "ESP32 Main" as MCU
participant "ESP32 CAM" as Cam

== Đặt mục tiêu ==
Operator -> Web: Chọn person_id từ danh sách
Web -> ROS: call /enrollment/set_target(person_id)
ROS -> Enroll: SetTrackingTarget(person_id)
Enroll -> Enroll: UPDATE tracking_target SET person_id = ?
Enroll --> ROS: success = true
ROS --> Web: SetTrackingTargetResponse

Tracker -> Enroll: poll current_target_id (DB hot-reload)
Enroll --> Tracker: current_target_id

== Bật Face Tracking ==
Operator -> Web: Toggle Face Tracking ON
Web -> Web: trackingEnabled = true (UI state)

== Vòng lặp tracking ==
loop Mỗi frame camera
    Cam -> ROS: /camera/image_raw
    MCU -> ROS: /scan
    ROS -> Tracker: /camera/image_raw + /camera_info + /scan
    Tracker -> Tracker: YOLOv8n body detection
    Tracker -> Tracker: InsightFace embedding + match DB
    Tracker -> Tracker: leg-pair LiDAR cluster, khớp bearing
    Tracker -> ROS: publish /tracked_persons (is_target, range_m, bearing_rad)
    ROS -> Ctrl: /tracked_persons
    ROS -> Web: /tracked_persons (overlay bbox)

    alt Có target
        Ctrl -> Ctrl: PID servo (50 Hz) + PID yaw + PID linear (10 Hz)
        Ctrl -> ROS: publish /servo_cmd (camera_pan_joint)
        Ctrl -> ROS: publish /cmd_vel
        ROS -> MCU: /servo_cmd + /cmd_vel
        MCU -> MCU: PID motor + servo PWM
    else Mất target
        Ctrl -> Ctrl: SEARCH_CONTINUE → SCAN → ROTATE
        Ctrl -> ROS: publish /servo_cmd (sweep) + /cmd_vel (rotate)
        ROS -> MCU: /servo_cmd + /cmd_vel
    end

    Ctrl -> ROS: publish /tracking_controller/status (5 Hz)
    ROS -> Web: /tracking_controller/status
    Web --> Operator: Hiển thị state + range + obstacle
end

== Tinh chỉnh PID (tùy chọn) ==
Operator -> Web: Chỉnh slider PID gain
Web -> ROS: call /tracking_controller_node/set_parameters
ROS -> Ctrl: SetParameters(pid_*_*)
Ctrl -> Ctrl: validate (>= 0), update PID kp/ki/kd
Ctrl --> ROS: SetParametersResult(successful=true)
ROS --> Web: result

== Tắt Face Tracking ==
Operator -> Web: Toggle Face Tracking OFF
Web -> ROS: publish /cmd_vel (zero) — đảm bảo robot dừng
Ctrl -> Ctrl: state = IDLE, reset PID
@enduml
```
