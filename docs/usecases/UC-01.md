## UC-01: Tạo lập bản đồ bằng di chuyển thủ công

### Mô tả use case

| Mục                            | Nội dung                                                                                                                                                                                                  |
| ------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Phụ thuộc                      | Không                                                                                                                                                                                                     |
| Mục đích                       | Người vận hành cần khảo sát một không gian mới mà robot chưa có bản đồ. PM cho phép người vận hành điều khiển robot di chuyển thủ công qua joystick/bàn phím, đồng thời xây dựng bản đồ 2D theo thời gian thực nhờ SLAM. |
| Mô tả                          | Người vận hành điều khiển robot di chuyển trong không gian bằng joystick ảo hoặc phím mũi tên, hệ thống sử dụng LiDAR + odometry để xây dựng bản đồ occupancy grid theo thời gian thực, sau đó lưu bản đồ khi hoàn tất. |
| Actor chính                    | Người vận hành (Operator)                                                                                                                                                                                 |
| Actor liên quan                | SLAM Toolbox (xử lý thuật toán SLAM), ESP32 firmware (điều khiển động cơ, đọc encoder, quét LiDAR)                                                                                                       |
| Tiền điều kiện                 | 1. Robot đã bật nguồn và kết nối WiFi <br> 2. Web dashboard đã kết nối rosbridge (status = connected) <br> 3. Hệ thống đang ở chế độ SLAM Mapping                                                        |
| Dãy lệnh thực hiện bình thường | 1. Người vận hành mở web dashboard và xác nhận kết nối ROS thành công <br> 2. Người vận hành chọn chế độ "Mapping" trên Mode Controller <br> 3. Người vận hành sử dụng joystick ảo hoặc phím mũi tên để điều khiển robot di chuyển <br> 4. Hệ thống publish /cmd_vel → ESP32 điều khiển động cơ <br> 5. ESP32 publish /scan (LiDAR) + /odom (encoder) → SLAM Toolbox xây dựng bản đồ <br> 6. Bản đồ occupancy grid được hiển thị real-time trên dashboard qua topic /map <br> 7. Người vận hành di chuyển robot khắp không gian cần quét <br> 8. Người vận hành nhấn "Save Map" và nhập tên bản đồ <br> 9. Hệ thống gọi service /slam_toolbox/serialize_map để lưu bản đồ |
| Hậu điều kiện (thành công)     | Bản đồ 2D (file .yaml + .pgm) được lưu trong thư mục maps, sẵn sàng sử dụng cho chế độ Navigation                                                                                                       |
| Hậu điều kiện (thất bại)       | Bản đồ không được lưu, dữ liệu SLAM trong bộ nhớ vẫn còn (có thể thử lưu lại). Robot dừng di chuyển an toàn.                                                                                             |
| Xử lý ngoại lệ                 | Mất kết nối rosbridge → Dashboard hiển thị lỗi, robot dừng (safety timeout) <br> Robot va chạm vật cản → Người vận hành dừng joystick, lùi lại <br> Lưu bản đồ thất bại → Hiển thị thông báo lỗi, cho phép thử lại |

### Lược đồ tuần tự

```plantuml
@startuml UC-01
title UC-01: Tạo lập bản đồ bằng di chuyển thủ công

actor "Người vận hành" as Operator
participant "Hệ thống" as System
participant "SLAM Toolbox" as SLAM

Operator -> System: cmd_vel(linear_x, angular_z)
System -> System: Điều khiển động cơ, thu thập LiDAR + odometry
System -> SLAM: /scan + /odom
SLAM -> SLAM: Xử lý SLAM, cập nhật bản đồ
SLAM --> System: /map (OccupancyGrid)
System --> Operator: Hiển thị bản đồ real-time

Operator -> System: saveMap(map_name)
alt Lưu thất bại
    System --> Operator: Error + "Failed to save map"
else Lưu thành công
    System -> SLAM: serialize_map(map_name)
    SLAM --> System: success
    System --> Operator: Success + "Map saved"
end
@enduml
```

### Phân rã thành phần PM

#### Controller: `DashboardWebApp`

- **Nhiệm vụ**: Nhận input từ người vận hành (joystick/keyboard), publish lệnh
  điều khiển qua rosbridge WebSocket.
- **Topic publish**: `/cmd_vel` (geometry_msgs/Twist)
- **Service call**: `/slam_toolbox/serialize_map` (slam_toolbox_msgs/SerializePoseGraph)
- **Input**: Joystick drag event hoặc Arrow key event → `Twist { linear.x, angular.z }`
- **Output thành công**: Robot di chuyển, bản đồ hiển thị real-time
- **Output lỗi**: Toast notification lỗi kết nối hoặc lỗi lưu bản đồ

#### UseCase: `ManualMappingUseCase`

- **Nhiệm vụ**: Orchestrate luồng điều khiển thủ công + SLAM mapping.
- **Input**: `Twist` — `{ linear.x: float, angular.z: float }`
- **Output**: `OccupancyGrid` (bản đồ real-time)
- **Gọi đến**:
    - `rosbridge.publish(/cmd_vel)` — gửi lệnh vận tốc đến robot
    - `rosbridge.subscribe(/map)` — nhận bản đồ cập nhật
    - `rosbridge.callService(/slam_toolbox/serialize_map)` — lưu bản đồ

#### Firmware: `ESP32 Main`

- **Nhiệm vụ**: Nhận /cmd_vel, điều khiển động cơ qua TB6612FNG, đọc encoder
  và LiDAR, publish /odom + /scan.
- **Subscribe**: `/cmd_vel` (geometry_msgs/Twist)
- **Publish**:
    - `/scan` (sensor_msgs/LaserScan) — dữ liệu LiDAR 360°
    - `/odom` (nav_msgs/Odometry) — odometry từ wheel encoder
    - `/imu/data_raw` (sensor_msgs/Imu) — dữ liệu IMU

#### Port: `SLAM Toolbox`

- **Nhiệm vụ**: Xử lý thuật toán SLAM, xây dựng bản đồ từ /scan + /odom.
- **Subscribe**: `/scan`, TF (odom → base_footprint)
- **Publish**: `/map` (nav_msgs/OccupancyGrid)
- **Service**: `/slam_toolbox/serialize_map` — lưu bản đồ ra file

#### Lược đồ tuần tự nội bộ PM

```plantuml
@startuml UC-01-internal
title UC-01: Tạo lập bản đồ thủ công - Internal Sequence

actor "Người vận hành" as Operator
participant "Web Dashboard" as Web
participant "rosbridge" as ROS
participant "ESP32" as MCU
participant "SLAM Toolbox" as SLAM
database "File System" as FS

== Di chuyển thủ công ==
Operator -> Web: Joystick drag / Arrow key
Web -> ROS: publish /cmd_vel (linear_x, angular_z)
ROS -> MCU: /cmd_vel
MCU -> MCU: PID motor control
MCU -> ROS: publish /scan (LaserScan)
MCU -> ROS: publish /odom (Odometry)
ROS -> SLAM: /scan + TF(odom→base_footprint)
SLAM -> SLAM: Graph-based SLAM processing
SLAM -> ROS: publish /map (OccupancyGrid)
ROS -> Web: /map
Web --> Operator: Render bản đồ real-time

== Lưu bản đồ ==
Operator -> Web: Click "Save Map" (map_name)
Web -> ROS: call /slam_toolbox/serialize_map(map_name)
ROS -> SLAM: serialize_map(map_name)
SLAM -> FS: Write .yaml + .pgm
FS --> SLAM: OK
SLAM --> ROS: result = 0 (success)
ROS --> Web: SaveMapResponse(result=0)
Web --> Operator: "Map saved successfully"
@enduml
```
